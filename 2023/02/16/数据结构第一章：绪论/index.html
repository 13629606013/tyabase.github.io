# 1.1 基本概念

1. 数据：计算机加工处理的对象

数据类型有两种：数值数据和非数值数据

（1）数值数据：包括整数、实数或者复数。主要用于工程与科学计算。

（2）非数值数据：包括字符、文字、图形、图象、语音等。用于情报检索、图形图象、人工智能、远程教育等诸多领域。

2. 数据元素（element）：是数据的基本单位，通常作为一个整体进行考虑和处理。

3. 数据项（item）：是构成数据元素的不可分割的最小单位，一个数据元素可由若干数据项组成。

4. 数据对象：是具有相同性质的数据元素的集合，是数据的一个子集。

5. 数据类型：是一个值的集合和定义在此集合上的一组操作的总称

6. 抽象数据类型（ADT）：是指一个数学模型及定义在该模型上的一组操作。

ADT 的定义仅取决于它的一组逻辑特性，而与其在计算机内部如何表示和实现无关。

7. 数据结构：

数据元素相互之间的关系称为结构。

数据结构是相互之间存在一种或多种特定关系的数据元素组成的集合，数据结构包括三个方面：

（1）逻辑结构：数据元素之间的逻辑关系，算法的设计取决于所选定的逻辑结构。

（2）存储结构：数据及数据之间的关系在计算机内的表示（也称映像，或物理结构），算法的实现依赖于所选用的物存储结构。

（3）运算：在数据结构上执行的运算，数据被使用的方式。

# 1.2 数据结构三要素

# 一、数据的四种逻辑结构

线性结构：数据元素之间存在一对一关系，一个前驱一个后继。

树形结构：数据元素之间存在一对多的关系。

图结构：数据元素之间存在多点对多点的关系。每个结点的前驱和后继的数目各有不同。

集合结构：结构之中的数据元素之间除了同属于一个集合以外没有其他关系。

四种逻辑结构也可以分为两类：线性结构和非线性结构。其中虽然树形结构有连线关系，但他并不是一对一关系，所以不属于线性结构，这点需要注意。

# 二、数据的四种存储结构

顺序结构

链接结构

以上两者是最基本的存储表示方式

索引结构

散列结构

下面我们来分别了解一下这四种存储结构

# 顺序存储结构

是将逻辑上相邻的数据元素存储在物理地址也相邻的存储单元中；元素之间的关系由存储单元的邻接关系来体现。

  • 优点：可实现随机存取，每个元素占用最少的存储空间；
  • 缺点：只能使用相邻的一整块存储单元，可能产生较多的外部碎片。

举个栗子：

例如， 由 4 个元素组成的线性数据结构（a0, a1, a2, a3），存储在某个连续的存储区内，设存储区的起始地址是 102，假定每个元素占 2 个存储单元。

那么就存在 Loc (ak) = 102 + 2 ×k

# 链接存储结构

不要求逻辑上相邻的数据元素在物理地址上也相邻；借助于指示元素存储地址的指针来表示元素之间的逻辑关系。

  • 优点：不会出现碎片现象，能充分利用所有存储单元

  • 缺点：每个元素因存储指针而占用额外的存储空间，且只能顺序存取。

# 索引存储结构

除数据元素存储在一组地址连续的内存空间外，还需建立一个索引表，索引表中的每一项称为索引项，索引项的一般形式是（关键字，下标）。

  • 优点：检索速度快

  • 缺点：增加附加的索引表后会占用较多的存储空间，另外，在增加和删除数据时要同步修改索引表，因而会花费较多的时间

# 散列存储结构

通过散列（哈希）函数和解决冲突的方法，将关键字散列在连续的有限的地址空间内，并将散列函数的值解释成关键字所在元素的存储地址。其特点是存取速度快，只能按关键字随机存取，不能顺序存取，也不能折半存取。

  • 优点：检索、增加和删除结点的速度都很快

  • 缺点：若散列函数不好，则可能出现元素存储单元的冲突，而解决冲突会增加时间和空间开销。

# 1.3 数据抽象和抽象数据类型

抽象：抽取事物的共同的和实质的东西，忽略其不是本质的细枝末节。抽象包括数据抽象和过程抽象。

还是举个栗子：

“学生”，这一概念是对学生群体的抽象，抽取了学生群体的共性，忽略了每个学生的特殊性。

在这里我们要学习一个新的概念，抽象数据类型：

抽象数据类型 (Abstract Data Type, ADT) 是一个数学模型以及在其上定义的运算集合。其最主要的两个特征是数据封装和信息隐蔽。

那什么又是数据封装和信息隐蔽呢？

定义：数据封装是指把数据和操纵数据的运算组合在一起的机制。信息隐蔽是指数据的使用者只需知道这些运算的定义（也称规范）便可访问数据，而无须了解数据的存储以及算法实现细节。

通过数据封装和信息隐蔽，可使数据的使用和实现相分离。

再举个栗子：C 语言中的整型 int 就是抽象数据类型。它的实现是隐蔽的，使用者只能通过整型上定义的一组运算对整型变量执行操作。

我们将抽象数据类型的定义格式如下：

ADT 抽象数据类型名

{

数据：

数据元素及其之间关系的定义

运算：

运算 1 (参数表)：运算功能描述

......

运算 n (参数表)：运算功能描述

}

在数据结构中，最常见的运算有以下几种

ℹ️创建运算：创建一个数据结构

ℹ️清除运算：删除数据结构中的全部元素

ℹ️插入运算（增）：在数据结构的指定位置插入一个新元素

ℹ️删除运算（删）：将数据结构中的某个元素删除

ℹ️搜索运算（查）：在数据结构中搜索满足一定条件的元素

ℹ️更新运算（改）：修改数据结构中的某个指定元素的值

ℹ️访问运算：访问数据结构中的某个元素

注意：我们将一种数据结构视为一个抽象数据类型，从规范和实现两方面来讨论数据结构。

  • C 语言的函数首部（声明）规范了 “做什么”
  • C 语言的函数体实现了 “怎么做”

[规范依赖于逻辑结构，实现依赖于存储结构].

因此，一个规范基于不同的存储结构会有不同的具体实现！

# 1.4 算法和算法分析

算法是对特定问题的求解步骤，它是指令的有限序列

  • 算法的五个特征：

（1）输入：有零个或者多个输入

（2）输出：至少产生一个输出（运算结果或提示信息）

（3）确定性：每一条指令都有确切的定义，没有二义性

（4）可行性：可以通过已经实现的基本运算执行有限次来实现

（5）有穷性：算法必须总能在执行有限步之后终止

  • 算法的性能标准

（1）正确性：在合理的数据输入下，算法可以在有限的时间内产生满足预先规定的功能和性能要求。

（2）可读性：一个好的算法应当思路清晰、简单明了。

（3）健壮性：一个好的算法应该在输入不合法的数据时，能做出适当处理，而不至于产生异常或者出现崩溃等严重后果。

（4）高效性：评价一个算法的效率主要包括时间和空间两个方面。

知道了算法的基本概念，接下来我们要学习算法的一个重要知识：时间复杂度

程序步：定义是在语法上或者语义上有意义的程序段，该程序段的执行时间 [与具体执行环境无关].{red}。

💁注意：一个算法时间花销与算法中语句的执行次数成正比例：算法中语句执行次数多，他的时间花销就多。一个算法中的语句在执行次数称为 [语句频度].{red}。

# 渐进时间复杂度

使用大 O 记号表示的算法的时间复杂度，称为算法的渐近时间复杂度。渐近时间复杂度也常简称为时间复杂度。

例如：O（1）表示常数时间，即算法只需要进行有限个程序步。

在大 O 记号下，可用程序步来估计算法的执行时间。同时在很多情况下，可以通过考察一个算法中的关键操作的执行次数来计算算法的渐近时间复杂度。

求解方法：

（1）如果没有循环语句，则为 O (1)

（2）有循环语句，则选择最深层的循环体，直接累计次数，得出与 n 之间的函数关系

这里又分为：循环主体中的变量与循环条件有关、无关这两种情况

（3）递归问题，使用公式进行递推，令 k 次后问题规模为 1，得出 k 与 n 的关系，代入公式计算

💁当然，具体还是得看习题。

# 空间复杂度

算法的空间复杂度是程序运行从开始到结束的存储量。一般按照最坏情况来分析。

程序运行所需的存储空间包括两部分：

（1）固定部分：主要包括程序代码、常量、简单变量、定长成分的结构变量所占的空间。

（2）可变部分：与算法在某次执行中处理的特定数据的大小和规模有关，如某调用函数中动态申请的数组空间。

算法原地工作是指算法所需的辅助空间为常量，即 O (1)
